# 聊天请求日志路径说明

## 日志文件路径

### 1. 主日志文件（包含所有聊天请求日志）

**路径格式**：`logs/yyyy/MM/dd/wechat-collector.log`

**示例**：
- 今天的日志：`logs/2026/01/07/wechat-collector.log`
- 昨天的日志：`logs/2026/01/06/wechat-collector.log`

**说明**：
- 日志按日期自动分目录存储
- 保留30天的日志
- 包含所有级别的日志（INFO、DEBUG、ERROR等）

### 2. 错误日志文件

**路径格式**：`logs/yyyy/MM/dd/error.log`

**示例**：
- 今天的错误日志：`logs/2026/01/07/error.log`

**说明**：
- 只包含 ERROR 级别的日志
- 如果聊天请求失败，错误信息会记录在这里

### 3. 控制台日志文件

**路径**：`logs/console.log`

**说明**：
- 包含应用启动时的控制台输出
- 包含所有标准输出和错误输出

## 查看日志的命令

### 查看今天的聊天请求日志

```bash
# 查看今天的日志（实时）
tail -f logs/$(date +%Y/%m/%d)/wechat-collector.log

# 查看今天的日志（最后100行）
tail -100 logs/$(date +%Y/%m/%d)/wechat-collector.log

# 查看今天的日志（全部）
cat logs/$(date +%Y/%m/%d)/wechat-collector.log
```

### 查看指定日期的日志

```bash
# 查看2026年1月7日的日志
tail -f logs/2026/01/07/wechat-collector.log

# 查看昨天的日志
tail -f logs/$(date -d yesterday +%Y/%m/%d)/wechat-collector.log
```

### 搜索聊天请求相关的日志

```bash
# 搜索包含 "GetChatData" 的日志（API原始响应）
grep -r "GetChatData" logs/

# 搜索包含 "Fetching chat data" 的日志（开始拉取）
grep -r "Fetching chat data" logs/

# 搜索包含 "Fetched.*messages" 的日志（拉取成功）
grep -r "Fetched.*messages" logs/

# 搜索包含 "Successfully saved" 的日志（保存成功）
grep -r "Successfully saved" logs/

# 搜索包含 "Failed to fetch chat data" 的日志（拉取失败）
grep -r "Failed to fetch chat data" logs/

# 搜索包含 "Error collecting messages" 的日志（收集错误）
grep -r "Error collecting messages" logs/
```

### 查看错误日志

```bash
# 查看今天的错误日志
tail -f logs/$(date +%Y/%m/%d)/error.log

# 查看所有错误日志
find logs -name "error.log" -exec tail -f {} \;
```

## 聊天请求相关的关键日志信息

### 1. 开始拉取消息

**日志级别**：DEBUG  
**日志内容**：`Fetching chat data from seq: {seq}, limit: {limit}`  
**位置**：`WeChatSdkService.fetchChatData()`  
**说明**：记录开始拉取消息，包含起始seq和拉取数量限制

### 2. API原始响应（重要）

**日志级别**：INFO  
**日志内容**：`WeChat GetChatData raw response: {jsonData}`  
**位置**：`WeChatSdkService.parseChatData()`  
**说明**：包含企业微信API返回的完整JSON响应，这是最详细的日志信息

**示例**：
```
2026-01-07 10:30:15.123 [pool-1-thread-1] INFO  c.w.c.s.WeChatSdkService - WeChat GetChatData raw response: {"errcode":0,"errmsg":"ok","chatdata":[{"seq":12345,"msgid":"xxx","publickey_ver":1,"encrypt_random_key":"xxx","encrypt_chat_msg":"xxx"},...]}
```

### 3. 拉取成功

**日志级别**：INFO  
**日志内容**：`Fetched {count} messages from WeChat`  
**位置**：`WeChatSdkService.fetchChatData()`  
**说明**：记录成功拉取的消息数量

### 4. 保存成功

**日志级别**：INFO  
**日志内容**：`Successfully saved {savedCount} new messages (total fetched: {totalCount})`  
**位置**：`MessageCollectorService.collectMessages()`  
**说明**：记录成功保存到数据库的消息数量

### 5. 没有新消息

**日志级别**：DEBUG  
**日志内容**：`No new messages to collect`  
**位置**：`MessageCollectorService.collectMessages()`  
**说明**：企业微信服务器上没有新消息（正常情况）

### 6. 拉取失败

**日志级别**：ERROR  
**日志内容**：`Failed to fetch chat data, error code: {ret}`  
**位置**：`WeChatSdkService.fetchChatData()`  
**说明**：SDK调用失败，ret为错误码

### 7. API返回错误

**日志级别**：ERROR  
**日志内容**：`API returned error: {errcode} - {errmsg}`  
**位置**：`WeChatSdkService.parseChatData()`  
**说明**：企业微信API返回了错误码和错误信息

### 8. 收集消息异常

**日志级别**：ERROR  
**日志内容**：`Error collecting messages`  
**位置**：`MessageCollectorService.collectMessages()`  
**说明**：消息收集过程中发生异常，会包含完整的异常堆栈

## 日志查看示例

### 示例1：查看最近一次聊天请求的完整日志

```bash
# 查看最后50行日志，包含最近的请求
tail -50 logs/$(date +%Y/%m/%d)/wechat-collector.log | grep -A 10 -B 5 "GetChatData\|Fetching chat data\|Fetched.*messages"
```

grep "Successfully saved" logs/$(date +%Y/%m/%d)/wechat-collector.log | wc -l

grep "Failed to fetch chat data\|Error collecting messages" logs/$(date +%Y/%m/%d)/wechat-collector.log | wc -l

```bash
# 实时查看聊天请求日志（过滤掉DEBUG日志）
tail -f logs/$(date +%Y/%m/%d)/wechat-collector.log | grep -E "GetChatData|Fetched|Successfully saved|Failed|Error"
```

### 示例4：查看API原始响应（调试用）

```bash
# 查看最近的API原始响应
grep "WeChat GetChatData raw response" logs/$(date +%Y/%m/%d)/wechat-collector.log | tail -1
```

## 日志配置说明

日志配置文件：`src/main/resources/logback.xml`

**关键配置**：
- 日志级别：INFO（DEBUG日志不会输出到文件，除非修改配置）
- 日志格式：`%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n`
- 日志保留：30天
- 日志大小限制：1GB

**注意**：
- `logger.debug()` 的日志默认不会输出到文件（因为日志级别是INFO）
- 如果需要查看DEBUG日志，需要修改 `logback.xml` 中的日志级别为DEBUG
